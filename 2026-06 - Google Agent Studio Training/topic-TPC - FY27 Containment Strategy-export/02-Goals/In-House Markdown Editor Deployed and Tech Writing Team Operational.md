## Success Criteria

- The in-house Markdown editor (built by Phil Raynes and Mackenzie) is deployed and in active use by the Technical Writing team
- Tech Writing team has been trained and is authoring content through the editor as their primary workflow
- Editor successfully writes structured Markdown to the repository that feeds the DPL/ETL pipeline
- At least one content type (e.g., release notes or user guide) has been fully produced through the new authoring workflow end-to-end
- MadCap Flare has been retired or decommissioned as the authoring tool
